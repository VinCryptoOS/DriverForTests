
Выход для примера

Запустить команду (log - файл, в который будет выводится вывод)
dotnet publish/Tests.dll &> log

Обратите внимание, что будет выдано одно предупреждение: "Incorrect task: Tests.TestIncorrect_1"
Так и должно быть.
При этом задача TestSingleThread_1 не создаётся автоматически, но и не вызывает предупреждение.
    Это связано с тем, что эта задача помечена атрибутом с флагом notAutomatic: true
    То есть её специально пометили как задачу, которую нужно вызывать только вручную

Задача TestSlowAndFastAndMedium_1 имитирует возникновение ошибки в тесте: информация об этом выводится при окончании тестирования
